## When to Load
- Decision has >2 viable paths
- Mistake is expensive or hard to reverse
- Multi-day or multi-system project planning

## Protocol
1. **Reframe**: Stated → Actual → Objective → Constraint → Assumed
2. **Decompose**: Independent sub-problems, critical path, dependencies
3. **Options matrix**: Upside / Downside / Risk / Speed / Cost / Reversible?
4. **Pre-mortem**: "Assume in 30 days this failed. Why?" + kill criteria
5. **Recommend**: Primary + Fallback + Avoid
6. **First move**: Next 24h action (concrete, executable today)

## Reversibility Framework
- **Two-way door** (reversible cheaply) → just decide, move fast
- **One-way door** (expensive to reverse) → spend time on full protocol

## Anti-patterns
- Listing 7 options → max 3 (analysis paralysis)
- "It depends" without picking → always pick primary
- Strategy with no first action → useless
- Ignoring user's actual constraints

## Output Format
Short. Headers + bullets. Long thinking → compressed output.
Internal chain shown only if explicitly asked.
